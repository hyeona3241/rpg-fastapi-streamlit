# 몬스터, 전투, 스킬 및 퀘스트 시스템 설계 문서

# 1. 개요

본 시스템은 Streamlit 기반의 RPG 테스트 애플리케이션에서 **몬스터와의 턴제 전투**, **스킬 사용**, **보상 지급**, **퀘스트 진행**을 구현하기 위한 설계이다.

실시간 액션 전투 대신 REST API와 데이터베이스 연동에 적합한 **턴제 전투 방식**을 채택하며, 캐릭터가 장착한 스킬을 이용하여 몬스터를 공격하고 처치한 후 아이템과 경험치를 획득한다.

획득한 아이템은 인벤토리에 저장되며, 이후 동적 크래프팅 시스템의 재료로 활용할 수 있다.

# 3. 몬스터 시스템

## 3.1 몬스터 정보

몬스터 정보는 데이터베이스에서 조회하여 표시한다.

| 정보    | 설명           |
| ----- | ------------ |
| 이름    | 몬스터 이름       |
| 설명    | 몬스터 설명       |
| HP    | 최대 체력        |
| 공격력   | 기본 공격력       |
| 방어력   | 기본 방어력       |
| 드롭 보상 | 처치 시 지급되는 보상 |
| 레벨    | 난이도 표현용      |

---

## 3.2 Streamlit 화면 구성

```text
┌─────────────────────────────┐
│ Slime                      │
│ HP : 30 / 30               │
│ ATK : 5                    │
│ DEF : 2                    │
│                             │
│ [ Fire Ball ]              │
│ [ Slash ]                  │
│ [ Ice Spear ]              │
└─────────────────────────────┘
```

---

## 3.3 전투 방식

실시간 전투가 아닌 턴제 버튼 기반 전투를 사용한다.

플레이어는 장착한 스킬 버튼을 눌러 공격한다.

공격 시

* 데미지 계산
* 몬스터 HP 감소
* 처치 여부 확인

을 수행한다.

---

# 4. 전투 세션

전투 시작 시 별도의 전투 상태를 생성한다.

## 목적

* 몬스터 원본 데이터 보호
* 현재 HP 저장
* 중복 보상 지급 방지

예시

```text
BattleSession

- battle_id
- char_id
- monster_id
- current_hp
- reward_claimed
- status
```

# 5. 스킬 시스템

## 5.1 장착 가능한 스킬

캐릭터는 보유한 스킬 중 일부만 장착할 수 있다.

예시

| 슬롯     | 스킬        |
| ------ | --------- |
| Slot 1 | Slash     |
| Slot 2 | Fire Ball |
| Slot 3 | Heal      |

전투에서는 장착된 스킬만 사용할 수 있다.

---

## 5.2 스킬 사용 조건

사용 전 확인한다.

* 현재 캐릭터가 보유한 스킬인지
* 장착된 스킬인지
* MP가 충분한지
* 쿨타임이 끝났는지

조건을 만족하면 사용 가능하다.

---

## 5.3 데미지 계산

예시

```text
Damage

= Character ATK
+ Skill Power
- Monster DEF
```

필요 시 속성 보정 및 추가 효과를 적용할 수 있다.

---

# 6. 몬스터 처치

몬스터 HP가 0 이하가 되면 처치 처리한다.

처리 과정

1. 처치 상태 변경
2. 경험치 지급
3. 드롭 보상 계산
4. 인벤토리 반영
5. 퀘스트 진행도 갱신

---

# 7. 보상 시스템

## 7.1 경험치

캐릭터에게 경험치를 지급한다.

```text
Character EXP
+ Reward EXP
```

레벨업 조건을 만족하면 레벨을 증가시킨다.

---

## 7.2 아이템

드롭 테이블을 이용하여 아이템을 지급한다.

획득한 아이템은 선택된 캐릭터의 인벤토리에 추가된다.

---

## 7.3 지급 순서

```text
몬스터 처치
      │
      ▼
Reward 조회
      │
      ▼
Drop 계산
      │
      ▼
Inventory 추가
      │
      ▼
Quest 진행도 갱신
```

---

# 8. 인벤토리 연동

획득한 아이템은 즉시 인벤토리에 반영된다.

예시

```text
Before

Slime Gel : 2

Monster Kill

After

Slime Gel : 3
```

획득한 재료는 이후 크래프팅에 사용할 수 있다.

---

# 9. NPC 및 퀘스트 시스템

## 9.1 NPC 기능

NPC는 플레이어에게 퀘스트를 제공한다.

퀘스트를 수락하면 CharacterQuest에 등록된다.

---

## 9.2 퀘스트 정보

| 정보     | 설명                 |
| ------ | ------------------ |
| 이름     | 퀘스트 이름             |
| 설명     | 퀘스트 내용             |
| 목표     | 처치 또는 수집 목표        |
| 현재 진행도 | 진행 수치              |
| 완료 여부  | Active / Completed |

---

## 9.3 퀘스트 화면

```text
Quest List

[✓] Slime Hunter
Progress : 3 / 10

[ ] Herb Collector
Progress : 1 / 5
```

---

# 10. 진행도 갱신

다음 이벤트 발생 시 자동 갱신한다.

* 몬스터 처치
* 아이템 획득
* 아이템 제작
* 특정 스킬 사용(확장 가능)

예시

```text
Kill Slime

Progress

4 / 10
→
5 / 10
```

---

# 11. 퀘스트 완료

목표 달성 시 완료 상태로 변경된다.

```text
ACTIVE

↓

COMPLETE
```

완료 시 보상을 지급한다.

* 경험치
* 아이템
* 해금 조건

# 12. 접근 권한

## 일반 사용자

가능

* 몬스터 전투
* 스킬 사용
* 보상 획득
* 퀘스트 수락

가능

* 몬스터 추가 및 수정
* 퀘스트 데이터 관리
* 드롭 테이블 관리

---

# 13. 예외 처리

## 전투

* 로그인하지 않은 사용자 전투 금지
* 현재 선택 캐릭터가 없으면 전투 불가
* 이미 종료된 전투 재사용 불가

---

## 스킬

* 장착하지 않은 스킬 사용 불가
* MP 부족 시 사용 불가
* 쿨타임 중 사용 불가

---

## 몬스터

* HP가 0인 몬스터 공격 불가
* 보상 중복 지급 금지

---

## 퀘스트

* 이미 완료한 퀘스트 중복 완료 불가
* 진행도는 목표 이상 증가하지 않음

---

## 보상

* 지급 실패 시 Rollback
* 아이템과 경험치는 함께 처리

---

# 14. 추천 API

```text
POST /battle/start
POST /battle/attack
POST /battle/end
GET  /battle/status
```

---

## 스킬

```text
GET  /characters/me/skills
POST /characters/me/skills/equip
POST /characters/me/skills/unequip
```

---

## 몬스터

```text
GET /monsters
GET /monsters/{monster_id}
```

---

## 퀘스트

```text
GET  /quests
POST /quests/{quest_id}/accept
GET  /quests/me
GET  /quests/{quest_id}/progress
POST /quests/{quest_id}/complete
```

# 16. 기대 효과

* 데이터베이스 기반 몬스터 및 퀘스트 관리
* 장착 스킬만을 이용한 직관적인 턴제 전투
* 전투 결과를 인벤토리와 경험치에 즉시 반영
* 퀘스트 진행도 자동 갱신 및 보상 지급
* 인벤토리와 동적 크래프팅 시스템까지 자연스럽게 연결되는 RPG 플레이 흐름 제공
