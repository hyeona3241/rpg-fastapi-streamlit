# 인벤토리 및 동적 크래프팅 시스템 설계 문서

# 1. 개요

인벤토리 및 크래프팅 시스템은 RPG 애플리케이션의 핵심 콘텐츠로, 캐릭터가 획득한 아이템을 관리하고 이를 활용하여 새로운 아이템을 제작하는 기능을 제공한다.

특히 본 프로젝트에서는 **동적 크래프팅(Dynamic Crafting)** 시스템을 도입하여, 기존에 정의된 레시피뿐만 아니라 AI(MLP + LLM)를 활용해 새로운 아이템과 레시피를 자동 생성하고 데이터베이스에 저장하는 기능을 제공한다.

---

# 2. 시스템 구성

```text
몬스터 처치 / 보상 획득
            │
            ▼
      인벤토리에 아이템 저장
            │
            ▼
     크래프팅 화면에서 재료 선택
            │
            ▼
      조합 방법(Method) 선택
            │
            ▼
     기존 레시피 존재 여부 확인
        │                 │
        │ 있음            │ 없음
        ▼                 ▼
 기존 결과 아이템 지급   AI(MLP + LLM) 생성
        │                 │
        │                 ▼
        │          새 아이템 생성
        │                 │
        │          새 레시피 저장
        └──────────┬──────┘
                   ▼
          재료 아이템 소비
                   ▼
      결과 아이템 인벤토리 추가
```

---

# 3. 인벤토리 시스템

## 3.1 기본 기능

각 캐릭터는 자신의 인벤토리를 가진다.

인벤토리에는 다음 정보가 저장된다.

| 항목     | 설명                                  |
| ------ | ----------------------------------- |
| 아이템 ID | 아이템 고유 번호                           |
| 아이템 이름 | 표시 이름                               |
| 수량     | 현재 보유 개수                            |
| 아이템 타입 | Material / Consumable / Equipment 등 |
| 설명     | 아이템 설명                              |
| 아이콘    | 이미지 또는 AI 생성 아이콘                    |

---

## 3.2 화면 구성

### 인벤토리 탭

* 보유 아이템 목록 조회
* 아이템 검색
* 아이템 정렬
* 아이템 상세 정보 확인

예시

```text
┌────────────────────────────────────┐
│ Name           Qty      Type       │
├────────────────────────────────────┤
│ Slime Gel       15      Material   │
│ Magic Herb       3      Material   │
│ Healing Potion   5      Consumable │
│ Iron Ore         9      Material   │
└────────────────────────────────────┘
```

---

## 3.3 접근 권한

* 로그인한 사용자만 조회 가능
* 선택된 캐릭터의 인벤토리만 조회 가능
* 다른 사용자의 인벤토리는 접근 불가

---

# 4. 아이템 시스템

## 4.1 아이템 구성 정보

모든 아이템은 다음 정보를 가진다.

| 정보            | 설명                                |
| ------------- | --------------------------------- |
| 이름            | 아이템 이름                            |
| 설명            | 아이템 설명                            |
| 타입            | Material / Equipment / Consumable |
| 서브 타입         | Weapon, Food 등                    |
| 아이콘           | 이미지 또는 AI 생성 아이콘                  |
| 속성(Attribute) | 독성, 식물성, 금속성 등                    |
| 효과(Effect)    | HP 회복, 공격력 증가 등                   |

---

## 4.2 AI 아이콘 생성

신규 아이템 생성 시 AI 기반 아이콘 생성 기능을 지원할 수 있다.

생성 절차

```text
아이템 이름
        │
        ▼
아이템 설명
        │
        ▼
AI 이미지 생성
        │
        ▼
생성 이미지 저장
        │
        ▼
Item.icon_url 등록
```

초기 버전에서는 이모지 또는 기본 아이콘으로 대체 가능하다.

---

# 5. 크래프팅 시스템

## 5.1 개요

플레이어는 인벤토리에 있는 재료를 이용하여 새로운 아이템을 제작할 수 있다.

크래프팅은 다음 요소를 사용한다.

* 재료 아이템
* 조합 방법(Method)
* AI 생성 모델

---

## 5.2 크래프팅 화면

```text
재료 1
[ Slime Gel ▼ ]

재료 2
[ Magic Herb ▼ ]

조합 방법
[ Boil ▼ ]

[ Craft ]
```

---

## 5.3 조합 방법(Method)

예시

| Method   |
| -------- |
| Mix      |
| Boil     |
| Bake     |
| Distill  |
| Compress |
| Infuse   |

동일한 재료라도 조합 방법이 다르면 다른 결과를 생성할 수 있다.

예시

```text
Slime Gel + Herb + Mix
→ Healing Slime

Slime Gel + Herb + Distill
→ Toxic Extract
```

---

# 6. 기존 레시피 조회

크래프팅 실행 시 먼저 데이터베이스를 조회한다.

```text
(재료1, 재료2, Method)
        │
        ▼
CraftingRecipe 검색
        │
 ┌──────┴──────┐
 │             │
있음          없음
 │             │
 ▼             ▼
기존 결과     AI 생성
```

이미 존재하는 레시피라면 기존 결과를 그대로 사용한다.

---

# 7. 신규 레시피 생성

레시피가 존재하지 않을 경우 AI를 이용하여 생성한다.

## 생성 과정

```text
재료 속성
        │
        ▼
MLP 효과 예측
        │
        ▼
LLM 이름 및 설명 생성
        │
        ▼
새 Item 생성
        │
        ▼
새 Recipe 저장
```

생성 이후 동일한 조합은 AI를 다시 호출하지 않는다.

---

# 8. 조합 성공 시 처리

조합 성공 시 다음 순서로 처리한다.

```text
1. 재료 존재 여부 확인

2. 레시피 확인

3. 결과 아이템 결정

4. 재료 수량 감소

5. 결과 아이템 생성(필요 시)

6. 결과 아이템 인벤토리 추가

7. Crafting Log 저장
```

---

# 9. 트랜잭션 처리

다음 과정은 하나의 트랜잭션으로 수행한다.

```text
재료 차감
        │
새 레시피 저장
        │
새 아이템 저장
        │
결과 지급
```

도중 하나라도 실패하면 전체 작업을 Rollback 한다.

---

# 10. 레시피 데이터베이스

추천 테이블

## CraftingRecipe

| 컬럼            | 설명       |
| ------------- | -------- |
| recipe_id     | 레시피 ID   |
| ingredient1   | 첫 번째 재료  |
| ingredient2   | 두 번째 재료  |
| method        | 조합 방법    |
| result_item   | 결과 아이템   |
| created_by_ai | AI 생성 여부 |
| created_at    | 생성 시각    |

---

## ItemAttribute

아이템의 물리적/화학적 속성을 저장한다.

예시

* Toxic
* Organic
* Metallic
* Plant
* Magical
* Cold
* Hot
* Electric
* Explosive

---

## ItemEffect

아이템의 실제 효과를 저장한다.

예시

| 효과               |
| ---------------- |
| HP 회복            |
| Poison           |
| Attack 증가        |
| Defense 증가       |
| Burn             |
| Freeze           |
| Shock            |
| Explosion Damage |

---

## CraftingLog

플레이어가 수행한 조합 기록을 저장한다.

| 정보     |
| ------ |
| 캐릭터    |
| 사용 재료  |
| 조합 방법  |
| 결과 아이템 |
| 생성 시각  |

---

# 11. 예외 처리

## 권한

* 로그인하지 않은 사용자는 크래프팅 불가
* 현재 선택된 캐릭터가 있어야 함
* 다른 캐릭터의 인벤토리 접근 불가

---

## 재료 검사

* 재료가 존재해야 함
* 필요한 수량 이상 보유해야 함
* 동일 아이템 2개 조합 시 수량 2 이상 필요

---

## 레시피 검사

* 기존 레시피 존재 시 AI 생성 금지
* 새 레시피만 AI 생성

---

## 인벤토리 처리

* 조합 성공 시 재료 감소
* 결과 아이템 지급
* 지급 실패 시 Rollback

---

## AI 생성 실패

* 생성 실패 메시지 출력
* 기존 데이터 변경 없음

---

# 15. 기대 효과

* 인벤토리와 데이터베이스 간 실시간 연동
* 재료 기반 동적 아이템 생성
