# 계정 및 캐릭터 관리 기능 설계 문서

## 1. 개요

본 애플리케이션은 Streamlit 기반 RPG 테스트 클라이언트와 FastAPI 기반 REST API 서버, MySQL 데이터베이스를 연동하여 RPG 게임 시나리오에 필요한 계정 및 캐릭터 관리 기능을 구현하는 것을 목표로 한다.

초기 프레임워크에는 로그인, 로그아웃, 캐릭터 목록 조회, 캐릭터 생성 기능이 포함되어 있으며, 이후 회원가입, 캐릭터 선택, 캐릭터 삭제, 공개 캐릭터 검색, 어드민 관리 기능 등을 확장한다.

---

## 2. 계정 기능 설계

### 2.1 회원가입

사용자는 새로운 계정을 생성할 수 있다.

#### 입력 정보

| 항목        | 설명              |
| --------- | --------------- |
| user_id   | 로그인에 사용할 고유 아이디 |
| user_name | 사용자 닉네임         |
| password  | 비밀번호            |

#### 처리 조건

* 동일한 `user_id`를 가진 계정은 생성할 수 없다.
* `user_id`, `user_name`, `password`는 비어 있을 수 없다.
* 계정 생성 성공 시 `User` 테이블에 새 레코드를 추가한다.
* 기본 권한은 일반 사용자 `USER`로 설정한다.

#### 예외 처리

| 상황            | 처리           |
| ------------- | ------------ |
| 이미 존재하는 ID 사용 | 회원가입 실패      |
| 필수 입력값 누락     | 경고 메시지 출력    |
| DB 저장 실패      | 서버 오류 메시지 반환 |

---

### 2.2 로그인

사용자는 생성된 계정 정보로 로그인할 수 있다.

#### 입력 정보

| 항목       | 설명      |
| -------- | ------- |
| user_id  | 사용자 아이디 |
| password | 비밀번호    |

#### 처리 조건

* 입력한 `user_id`가 DB에 존재하는지 확인한다.
* 비밀번호가 일치하는지 확인한다.
* 로그인 성공 시 서버는 `session_id` 쿠키를 발급한다.
* Streamlit 클라이언트는 발급받은 쿠키를 저장하고 이후 요청에 함께 전달한다.

#### 예외 처리

| 상황         | 처리                        |
| ---------- | ------------------------- |
| 존재하지 않는 ID | 로그인 실패                    |
| 비밀번호 불일치   | 로그인 실패                    |
| 이미 로그인된 상태 | 기존 로그인 정보 유지 또는 안내 메시지 출력 |
| 서버 연결 실패   | FastAPI 서버 실행 여부 안내       |

---

### 2.3 로그아웃

로그인한 사용자는 로그아웃할 수 있다.

#### 처리 조건

* 서버의 세션 저장소에서 해당 `session_id`를 제거한다.
* 클라이언트의 로그인 상태, 사용자 ID, 쿠키 정보를 초기화한다.

#### 예외 처리

| 상황            | 처리                        |
| ------------- | ------------------------- |
| 이미 세션이 만료된 상태 | 클라이언트 상태만 초기화             |
| 서버 연결 실패      | 클라이언트 상태는 초기화하고 오류 메시지 출력 |

---

### 2.4 내 계정 정보 조회

로그인한 사용자는 자신의 계정 정보를 확인할 수 있다.

#### 표시 정보

| 항목              | 설명       |
| --------------- | -------- |
| user_id         | 사용자 아이디  |
| user_name       | 사용자 닉네임  |
| role            | 사용자 권한   |
| character_count | 보유 캐릭터 수 |

#### 접근 제한

* 로그인하지 않은 사용자는 조회할 수 없다.
* 일반 사용자는 다른 계정 정보를 직접 조회할 수 없다.

---

### 2.5 비밀번호 변경

사용자는 자신의 비밀번호를 변경할 수 있다.

#### 입력 정보

| 항목               | 설명        |
| ---------------- | --------- |
| current_password | 현재 비밀번호   |
| new_password     | 새 비밀번호    |
| confirm_password | 새 비밀번호 확인 |

#### 처리 조건

* 현재 비밀번호가 일치해야 한다.
* 새 비밀번호와 확인 비밀번호가 일치해야 한다.
* 변경 성공 시 `User` 테이블의 password 값을 수정한다.

#### 예외 처리

| 상황            | 처리               |
| ------------- | ---------------- |
| 현재 비밀번호 불일치   | 변경 실패            |
| 새 비밀번호 확인 불일치 | 변경 실패            |
| 로그인하지 않은 접근   | 401 Unauthorized |

---

### 2.6 계정 삭제

사용자는 자신의 계정을 삭제할 수 있다.

#### 처리 조건

* 로그인한 사용자 본인만 삭제할 수 있다.
* 계정 삭제 시 해당 계정의 캐릭터, 인벤토리, 퀘스트 진행 정보 등 연결 데이터 처리 방식을 결정해야 한다.
* 구현 난이도를 낮추기 위해 초기 버전에서는 캐릭터가 존재할 경우 계정 삭제를 제한할 수 있다.

#### 예외 처리

| 상황               | 처리               |
| ---------------- | ---------------- |
| 로그인하지 않은 접근      | 401 Unauthorized |
| 다른 계정 삭제 시도      | 403 Forbidden    |
| 연결 데이터로 인한 삭제 실패 | 삭제 불가 안내         |

## 3. 캐릭터 기능 설계

### 3.1 내 캐릭터 목록 조회

로그인한 사용자는 자신이 생성한 캐릭터 목록을 조회할 수 있다.

#### 표시 정보

| 항목             | 설명        |
| -------------- | --------- |
| actor_id       | 캐릭터 고유 ID |
| character_name | 캐릭터 이름    |
| level          | 캐릭터 레벨    |
| exp            | 경험치       |
| active         | 현재 선택 여부  |
| is_public      | 공개 여부     |

#### 접근 제한

* 로그인한 계정의 캐릭터만 조회할 수 있다.
* 요청 URL에 다른 사용자의 ID를 입력하더라도 세션 정보와 일치하지 않으면 차단한다.

---

### 3.2 캐릭터 생성

로그인한 사용자는 새 캐릭터를 생성할 수 있다.

#### 입력 정보

| 항목             | 설명         |
| -------------- | ---------- |
| character_name | 생성할 캐릭터 이름 |

#### 처리 조건

* `Actor` 테이블에 먼저 레코드를 생성하여 `actor_id`를 발급받는다.
* 발급받은 `actor_id`를 사용해 `Character` 테이블에 캐릭터를 생성한다.
* 기본 레벨은 1, 경험치는 0으로 설정한다.
* 기본적으로 `active = false`, `is_public = false`로 설정한다.

#### 예외 처리

| 상황                | 처리               |
| ----------------- | ---------------- |
| 로그인하지 않은 접근       | 401 Unauthorized |
| 캐릭터 이름 미입력        | 생성 실패            |
| 같은 계정 내 캐릭터 이름 중복 | 생성 실패            |
| 최대 캐릭터 수 초과       | 생성 실패            |

---

### 3.3 캐릭터 선택

사용자는 보유 캐릭터 중 하나를 현재 플레이 캐릭터로 선택할 수 있다.

#### 처리 조건

* 선택한 캐릭터가 로그인한 사용자의 캐릭터인지 확인한다.
* 같은 계정의 기존 `active = true` 캐릭터를 모두 `false`로 변경한다.
* 선택한 캐릭터만 `active = true`로 설정한다.

#### 예외 처리

| 상황                | 처리               |
| ----------------- | ---------------- |
| 존재하지 않는 캐릭터 선택    | 404 Not Found    |
| 다른 사용자의 캐릭터 선택 시도 | 403 Forbidden    |
| 로그인하지 않은 접근       | 401 Unauthorized |

---

### 3.4 현재 선택 캐릭터 조회

로그인한 사용자는 현재 선택된 캐릭터 정보를 조회할 수 있다.

#### 사용 목적

현재 선택 캐릭터는 이후 RPG 기능의 기준이 된다.

* 인벤토리 조회
* 아이템 사용
* 장비 장착
* 퀘스트 수락
* 크래프팅 실행
* 보상 지급

#### 예외 처리

| 상황          | 처리               |
| ----------- | ---------------- |
| 선택된 캐릭터 없음  | 캐릭터 선택 안내        |
| 로그인하지 않은 접근 | 401 Unauthorized |

---

### 3.5 캐릭터 삭제

사용자는 자신의 캐릭터를 삭제할 수 있다.

#### 처리 조건

* 삭제 대상 캐릭터가 로그인한 사용자의 캐릭터인지 확인한다.
* 캐릭터 삭제 시 연결된 인벤토리, 장비, 퀘스트 진행 정보 등을 함께 처리해야 한다.
* 초기 구현에서는 연결 데이터가 있는 경우 삭제를 제한하거나, 관련 데이터를 먼저 삭제한 뒤 캐릭터를 삭제한다.

#### 예외 처리

| 상황                | 처리                        |
| ----------------- | ------------------------- |
| 다른 사용자의 캐릭터 삭제 시도 | 403 Forbidden             |
| 존재하지 않는 캐릭터 삭제    | 404 Not Found             |
| active 캐릭터 삭제     | 삭제 후 active 캐릭터 없음 상태로 전환 |
| 연결 데이터로 삭제 실패     | 삭제 불가 안내                  |

---

### 3.6 캐릭터 공개/비공개 설정

사용자는 자신의 캐릭터를 다른 플레이어에게 공개할지 결정할 수 있다.

#### 추가 필요 컬럼

```sql
ALTER TABLE `Character`
ADD COLUMN is_public BOOLEAN NOT NULL DEFAULT FALSE;
```

#### 처리 조건

* `is_public = true`인 캐릭터만 공개 검색 결과에 표시한다.
* 공개 정보는 기본 정보로 제한한다.
* 인벤토리, 장비 상세, 퀘스트 진행 상황 등 민감한 플레이 데이터는 공개하지 않는다.

#### 공개 가능 정보

| 항목                     | 설명           |
| ---------------------- | ------------ |
| character_name         | 캐릭터 이름       |
| level                  | 레벨           |
| job                    | 현재 직업        |
| specimen               | 종족 또는 기원     |
| public_profile_message | 소개 문구, 선택 사항 |

## 4. 공개 캐릭터 검색 기능

다른 사용자는 공개 상태의 캐릭터를 검색할 수 있다.

### 검색 조건

| 조건     | 설명           |
| ------ | ------------ |
| 캐릭터 이름 | 이름 일부 검색     |
| 레벨     | 최소/최대 레벨 필터  |
| 직업     | 특정 직업 캐릭터 검색 |
| 공개 여부  | 공개 캐릭터만 검색   |

### 처리 조건

* `is_public = true`인 캐릭터만 조회한다.
* 일반 사용자는 비공개 캐릭터를 검색할 수 없다.
* 검색 결과에는 기본 정보만 표시한다.

### 예외 처리

| 상황            | 처리                        |
| ------------- | ------------------------- |
| 검색 결과 없음      | 결과 없음 메시지 출력              |
| 비공개 캐릭터 접근 시도 | 403 Forbidden 또는 검색 결과 제외 |
| 잘못된 필터값 입력    | 입력값 검증 후 안내               |

## 5. 어드민 기능 설계

어드민 계정은 전체 계정과 캐릭터 데이터를 관리할 수 있다.

### 추가 필요 컬럼

```sql
ALTER TABLE `User`
ADD COLUMN role VARCHAR(20) NOT NULL DEFAULT 'USER';
```

### 권한 종류

| role  | 설명     |
| ----- | ------ |
| USER  | 일반 사용자 |
| ADMIN | 관리자    |

### 어드민 기능 목록

| 기능            | 설명                     |
| ------------- | ---------------------- |
| 전체 유저 목록 조회   | 모든 계정 정보 조회            |
| 유저 검색         | ID, 닉네임 기준 검색          |
| 전체 캐릭터 목록 조회  | 공개/비공개와 관계없이 전체 캐릭터 조회 |
| 캐릭터 검색        | 이름, 레벨, 직업, 소유자 기준 검색  |
| 특정 유저의 캐릭터 조회 | 관리 목적의 상세 조회           |
| 아이템 목록 관리     | 아이템 생성, 수정, 삭제         |
| 크래프팅 레시피 조회   | 등록된 조합식 전체 확인          |

### 접근 제한

* `role = ADMIN`인 계정만 어드민 화면에 접근할 수 있다.
* 일반 사용자가 어드민 API를 호출하면 `403 Forbidden`을 반환한다.

## 6. 접근 권한 및 예외 처리 설계

### 6.1 인증이 필요한 기능

다음 기능은 로그인한 사용자만 접근할 수 있다.

* 내 계정 정보 조회
* 비밀번호 변경
* 계정 삭제
* 내 캐릭터 목록 조회
* 캐릭터 생성
* 캐릭터 선택
* 캐릭터 삭제
* 캐릭터 공개/비공개 설정
* 인벤토리 조회
* 아이템 사용
* 퀘스트 수락
* 크래프팅 실행

로그인하지 않은 사용자가 접근하면 `401 Unauthorized`를 반환한다.

---

### 6.2 소유자 검사가 필요한 기능

다음 기능은 요청 대상 데이터가 현재 로그인한 사용자의 것인지 확인해야 한다.

* 특정 캐릭터 조회
* 캐릭터 선택
* 캐릭터 삭제
* 캐릭터 공개/비공개 변경
* 인벤토리 조회
* 아이템 사용
* 장비 장착
* 퀘스트 진행
* 크래프팅 결과 지급

다른 사용자의 데이터에 접근하려는 경우 `403 Forbidden`을 반환한다.

---

### 6.3 어드민 권한이 필요한 기능

다음 기능은 어드민만 사용할 수 있다.

* 전체 유저 조회
* 전체 캐릭터 조회
* 비공개 캐릭터 포함 검색
* 아이템 생성/수정/삭제
* 전체 크래프팅 레시피 관리
* 테스트 데이터 관리

일반 사용자가 접근하면 `403 Forbidden`을 반환한다.

---

## 8. 추천 API 설계

### Auth API

| Method | Endpoint       | 설명              |
| ------ | -------------- | --------------- |
| POST   | `/login`       | 로그인             |
| POST   | `/logout`      | 로그아웃            |
| GET    | `/me`          | 현재 로그인 계정 정보 조회 |
| PATCH  | `/me/password` | 비밀번호 변경         |

---

### User API

| Method | Endpoint              | 설명               |
| ------ | --------------------- | ---------------- |
| POST   | `/users`              | 회원가입             |
| DELETE | `/users/me`           | 내 계정 삭제          |
| GET    | `/admin/users`        | 전체 유저 조회, 어드민 전용 |
| GET    | `/admin/users/search` | 유저 검색, 어드민 전용    |

---

### Character API

| Method | Endpoint                           | 설명                |
| ------ | ---------------------------------- | ----------------- |
| GET    | `/characters/me`                   | 내 캐릭터 목록 조회       |
| POST   | `/characters`                      | 캐릭터 생성            |
| PATCH  | `/characters/{char_id}/select`     | 캐릭터 선택            |
| DELETE | `/characters/{char_id}`            | 캐릭터 삭제            |
| PATCH  | `/characters/{char_id}/visibility` | 공개/비공개 설정         |
| GET    | `/characters/public/search`        | 공개 캐릭터 검색         |
| GET    | `/admin/characters`                | 전체 캐릭터 조회, 어드민 전용 |

## 9. 구현 우선순위

### 1단계: 계정 기본 기능

1. 회원가입
2. 로그인
3. 로그아웃
4. 내 계정 정보 조회

### 2단계: 캐릭터 기본 기능

1. 내 캐릭터 목록 조회
2. 캐릭터 생성
3. 캐릭터 선택
4. 현재 선택 캐릭터 조회

### 3단계: 캐릭터 관리 기능

1. 캐릭터 삭제
2. 공개/비공개 설정
3. 공개 캐릭터 검색

### 4단계: 권한 기능

1. User 테이블에 role 컬럼 추가
2. 어드민 계정 구분
3. 어드민 전용 유저/캐릭터 조회

### 5단계: RPG 기능 연결

1. 선택 캐릭터 기준 인벤토리 조회
2. 아이템 획득 및 사용
3. 퀘스트 수락 및 완료
4. 크래프팅 실행 및 결과 저장
