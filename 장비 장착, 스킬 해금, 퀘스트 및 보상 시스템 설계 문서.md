# 장비 장착, 스킬 해금, 퀘스트 및 보상 시스템 설계 문서

# 1. 개요

본 문서는 RPG 애플리케이션에서 캐릭터 성장과 플레이 흐름을 완성하기 위한 **장비 장착**, **스킬 해금**, **퀘스트 내용**, **보상 지급** 시스템을 설계한다.

이 기능들은 캐릭터, 인벤토리, 몬스터 전투, 크래프팅 시스템과 연결되어 RPG 시나리오의 흐름을 구성한다.

---

# 2. 전체 플레이 흐름

```text
회원가입 / 로그인
        │
        ▼
캐릭터 생성 및 선택
        │
        ▼
기본 스킬 확인
        │
        ▼
NPC 퀘스트 수락
        │
        ▼
몬스터 전투
        │
        ▼
아이템 및 경험치 획득
        │
        ▼
인벤토리 확인
        │
        ▼
장비 장착
        │
        ▼
스탯 변화 확인
        │
        ▼
크래프팅
        │
        ▼
퀘스트 완료
        │
        ▼
보상 획득
        │
        ▼
레벨업 및 신규 스킬 해금
```

# 3. 장비 장착 시스템

## 3.1 개요

장비 장착 시스템은 인벤토리에 보유한 장비 아이템을 캐릭터의 장착 슬롯에 등록하고, 장비의 보너스 스탯을 캐릭터 능력치에 반영하는 기능이다.

---

## 3.2 장비 아이템 조건

장착 가능한 아이템은 `Item.type`이 `equipment`인 아이템으로 제한한다.

예시

| Item Type  | 장착 가능 여부 |
| ---------- | -------- |
| equipment  | 가능       |
| consumable | 불가       |
| material   | 불가       |
| others     | 불가       |

---

## 3.3 장비 슬롯

초기 구현에서는 다음 장비 슬롯만 사용한다.

| 슬롯     | 설명    |
| ------ | ----- |
| weapon | 무기    |
| head   | 머리 장비 |
| armor  | 방어구   |

이후 필요에 따라 `gloves`, `boots`, `accessory` 등을 추가할 수 있다.

---

## 3.4 장비 장착 흐름

```text
인벤토리에서 아이템 선택
        │
        ▼
장비 타입 검사
        │
        ▼
장착 부위 검사
        │
        ▼
기존 장비 해제
        │
        ▼
새 장비 장착
        │
        ▼
캐릭터 스탯 재계산
```

---

## 3.5 장비 스탯 반영

장비의 보너스 스탯은 `ItemBonusStat`을 기준으로 적용한다.

예시

```text
캐릭터 기본 ATK : 10
낡은 검 ATK +5

최종 ATK = 15
```

---

## 3.6 예외 처리

| 상황                  | 처리                 |
| ------------------- | ------------------ |
| 인벤토리에 없는 아이템 장착 시도  | 실패                 |
| 다른 캐릭터의 아이템 장착 시도   | 실패                 |
| 장비 타입이 아닌 아이템 장착 시도 | 실패                 |
| 슬롯과 맞지 않는 아이템 장착 시도 | 실패                 |
| 이미 장착 중인 슬롯         | 기존 장비 해제 후 새 장비 장착 |

---

# 4. 스킬 해금 시스템

## 4.1 개요

스킬 해금 시스템은 캐릭터가 성장함에 따라 새로운 스킬을 사용할 수 있게 하는 기능이다.

초기 구현에서는 구현 난이도를 낮추기 위해 **레벨 조건 기반 자동 해금 방식**을 사용한다.

---

## 4.2 스킬 해금 방식

### 추천 방식: 자동 해금

```text
몬스터 처치
        │
        ▼
경험치 획득
        │
        ▼
레벨업 검사
        │
        ▼
스킬 해금 조건 검사
        │
        ▼
조건 만족 스킬 자동 지급
```

---

## 4.3 해금 조건

스킬은 다음 조건에 따라 해금될 수 있다.

| 조건     | 설명                        |
| ------ | ------------------------- |
| 레벨 조건  | 특정 레벨 이상이면 해금             |
| 직업 조건  | 특정 직업을 보유하면 해금            |
| 종족 조건  | 특정 종족 또는 종족 비율 조건 만족 시 해금 |
| 퀘스트 조건 | 특정 퀘스트 완료 시 해금            |

초기 구현에서는 **레벨 조건**만 우선 사용한다.

---

## 4.4 스킬 장착

해금된 스킬은 모두 전투에서 바로 사용하는 것이 아니라, 일부만 장착할 수 있도록 한다.

| 항목       | 내용                 |
| -------- | ------------------ |
| 장착 가능 개수 | 최대 3개              |
| 사용 가능 조건 | 장착된 스킬만 전투에서 사용 가능 |
| 변경 가능 위치 | 스킬 관리 화면           |

---

## 4.5 예외 처리

| 상황               | 처리      |
| ---------------- | ------- |
| 해금되지 않은 스킬 장착 시도 | 실패      |
| 다른 캐릭터의 스킬 장착 시도 | 실패      |
| 장착 슬롯 초과         | 실패      |
| 이미 장착된 스킬 재장착    | 중복 방지   |
| 장착 해제 후 슬롯 정리    | 빈 슬롯 처리 |

---

# 5. 퀘스트 시스템

## 5.1 개요

퀘스트 시스템은 NPC를 통해 수락하고, 전투/아이템 획득/크래프팅 등의 행동을 통해 진행도를 갱신하는 기능이다.

Streamlit 기반 구현이므로 복잡한 스토리 연출보다는 **명확한 목표 기반 퀘스트**를 사용한다.

---

## 5.2 퀘스트 종류

초기 구현에서는 다음 3가지 퀘스트를 사용한다.

| 퀘스트 타입 | 설명                    | 예시            |
| ------ | --------------------- | ------------- |
| 몬스터 처치 | 특정 몬스터를 일정 수 처치       | 슬라임 3마리 처치    |
| 아이템 수집 | 특정 아이템을 일정 수 보유 또는 제출 | 슬라임 젤 2개 가져오기 |
| 크래프팅   | 특정 아이템을 제작            | 회복 물약 1개 제작   |

---

## 5.3 퀘스트 수락 흐름

```text
NPC 목록 조회
        │
        ▼
NPC 선택
        │
        ▼
제공 퀘스트 조회
        │
        ▼
퀘스트 상세 확인
        │
        ▼
퀘스트 수락
        │
        ▼
CharacterQuest 등록
```

---

## 5.4 퀘스트 진행도 갱신

다음 이벤트 발생 시 진행도를 갱신한다.

| 이벤트    | 진행도 갱신 대상  |
| ------ | ---------- |
| 몬스터 처치 | 몬스터 처치 퀘스트 |
| 아이템 획득 | 아이템 수집 퀘스트 |
| 아이템 제작 | 크래프팅 퀘스트   |

---

## 5.5 퀘스트 완료 흐름

```text
진행도 목표 달성
        │
        ▼
완료 가능 상태 표시
        │
        ▼
완료 버튼 클릭
        │
        ▼
보상 지급
        │
        ▼
CharacterQuest 상태 completed 변경
```

---

## 5.6 예외 처리

| 상황                   | 처리          |
| -------------------- | ----------- |
| 이미 수락한 퀘스트 재수락       | 실패          |
| 이미 완료한 퀘스트 재완료       | 실패          |
| 완료 조건 미달성 상태에서 완료 시도 | 실패          |
| 다른 캐릭터의 퀘스트 접근       | 실패          |
| 보상 지급 실패             | 전체 Rollback |

---

# 6. 보상 시스템

## 6.1 개요

보상 시스템은 몬스터 처치, 퀘스트 완료, 크래프팅 결과 등에 따라 캐릭터에게 경험치나 아이템을 지급하는 기능이다.

---

## 6.2 보상 종류

| 보상    | 설명             |
| ----- | -------------- |
| 경험치   | 캐릭터 EXP 증가     |
| 아이템   | 인벤토리에 아이템 추가   |
| 장비    | 장착 가능한 아이템 지급  |
| 스킬    | 특정 스킬 해금       |
| 해금 조건 | 새로운 퀘스트나 직업 해금 |

초기 구현에서는 **경험치 + 아이템 보상**을 우선 적용한다.

---

## 6.3 보상 지급 흐름

```text
보상 조건 만족
        │
        ▼
Reward 조회
        │
        ▼
RewardExp 지급
        │
        ▼
RewardItem 지급
        │
        ▼
레벨업 검사
        │
        ▼
스킬 해금 검사
        │
        ▼
퀘스트 상태 갱신
```

---

## 6.4 레벨업 처리

캐릭터의 경험치가 현재 레벨의 필요 경험치를 넘으면 레벨을 증가시킨다.

```text
현재 EXP + 보상 EXP >= 다음 레벨 필요 EXP
        │
        ▼
Level + 1
        │
        ▼
남은 EXP 반영
        │
        ▼
레벨 기반 스탯 갱신
        │
        ▼
스킬 해금 검사
```

# 7. 데이터베이스 설계 보완

## 7.1 장비 장착

기존 `CharacterEquipment` 테이블을 사용한다.

필요한 관계

```text
Character
    │
    ▼
CharacterEquipment
    │
    ▼
InventoryItem
    │
    ▼
Item
```

---

## 7.2 스킬 장착 테이블 추가

보유 스킬과 장착 스킬을 분리하기 위해 추가 테이블을 사용한다.

```sql
CREATE TABLE EquippedSkill (
    char_id INT NOT NULL,
    slot_no INT NOT NULL,
    skill_id INT NOT NULL,
    PRIMARY KEY (char_id, slot_no),
    FOREIGN KEY (char_id) REFERENCES `Character`(actor_id),
    FOREIGN KEY (skill_id) REFERENCES Skill(id)
);
```

---

## 7.3 퀘스트 목표 테이블 추가

현재 `Quest`에는 기본 정보만 있으므로, 퀘스트 목표를 명확히 표현하기 위한 테이블을 추가한다.

```sql
CREATE TABLE QuestObjective (
    quest_id INT NOT NULL,
    objective_type VARCHAR(50) NOT NULL,
    target_id INT NOT NULL,
    required_count INT NOT NULL,
    PRIMARY KEY (quest_id, objective_type, target_id),
    FOREIGN KEY (quest_id) REFERENCES Quest(id)
);
```

예시

| quest_id | objective_type | target_id | required_count |
| -------- | -------------- | --------: | -------------: |
| 1        | KILL_MONSTER   |         3 |              5 |
| 2        | COLLECT_ITEM   |        10 |              3 |
| 3        | CRAFT_ITEM     |        20 |              1 |

---

## 7.4 퀘스트 진행도 테이블 추가

`CharacterQuest.current_step`만으로는 세부 진행도 표현이 부족하므로 진행도 테이블을 추가한다.

```sql
CREATE TABLE CharacterQuestProgress (
    char_id INT NOT NULL,
    quest_id INT NOT NULL,
    objective_type VARCHAR(50) NOT NULL,
    target_id INT NOT NULL,
    current_count INT NOT NULL DEFAULT 0,
    PRIMARY KEY (char_id, quest_id, objective_type, target_id),
    FOREIGN KEY (char_id) REFERENCES `Character`(actor_id),
    FOREIGN KEY (quest_id) REFERENCES Quest(id)
);
```
