# 캐릭터 시스템 설계 문서

# 1. 개요

캐릭터 시스템은 RPG 애플리케이션의 핵심 기능으로, 계정에 종속되어 생성 및 관리된다.
각 캐릭터는 고유한 종족, 직업, 레벨, 스탯, 스킬, 인벤토리 등의 정보를 가지며, 이후 퀘스트, 크래프팅, 아이템 사용 등 모든 게임 기능의 주체가 된다.

본 시스템은 데이터베이스에 저장된 정보를 기반으로 캐릭터를 생성하고, 선택된 캐릭터를 중심으로 다양한 기능을 수행할 수 있도록 설계한다.

---

# 2. 캐릭터 생성

## 2.1 생성 절차

캐릭터 생성 시 다음 순서로 데이터가 생성된다.

```text
계정 로그인
        │
        ▼
캐릭터 이름 입력
        │
        ▼
종족 선택 및 비율 설정
        │
        ▼
Actor 생성
        │
        ▼
Character 생성
        │
        ▼
CharacterSpecimen 생성
        │
        ▼
기본 직업 부여
        │
        ▼
초기 스탯 계산
        │
        ▼
기본 스킬 지급
        │
        ▼
기본 인벤토리 생성
```

---

## 2.2 입력 정보

| 항목     | 설명                  |
| ------ | ------------------- |
| 캐릭터 이름 | 플레이어가 사용할 이름        |
| 종족 정보  | 하나 이상의 종족 선택        |
| 종족 비율  | 선택한 종족별 비율(총합 100%) |

---

## 2.3 기본 초기값

| 항목      | 초기값                  |
| ------- | -------------------- |
| Level   | 1                    |
| Exp     | 0                    |
| Active  | False                |
| 공개 여부   | False                |
| 기본 직업   | Beginner (또는 Novice) |
| 기본 인벤토리 | 자동 생성                |
| 기본 스킬   | 종족 및 직업 기반 자동 지급     |

---

# 3. 종족 시스템

## 3.1 단일 및 복합 종족 지원

캐릭터는 하나 이상의 종족으로 구성될 수 있다.

예시

| 종족    | 비율   |
| ----- | ---- |
| Human | 100% |

또는

| 종족    | 비율  |
| ----- | --- |
| Human | 50% |
| Elf   | 30% |
| Orc   | 20% |

복합 종족은 `CharacterSpecimen` 테이블에 여러 행으로 저장된다.

---

## 3.2 Streamlit UI 설계

캐릭터 생성 화면에서 다음과 같은 UI를 제공한다.

```
종족 선택

☑ Human
☑ Elf
☐ Orc

------------------------

Human    [=====50%=====]
Elf      [===30%=======]
Orc      [==20%========]

총합 : 100%
```

사용자는 슬라이더를 이용하여 각 종족의 비율을 조절할 수 있다.

---

## 3.3 검증 조건

### 허용

* 종족 1개 이상 선택
* 비율 총합 = 100%

### 제한

* 총합이 100%가 아니면 생성 불가
* 존재하지 않는 종족 선택 불가
* 생성 이후 종족 변경 불가

---

# 4. 종족 기반 스탯 계산

최종 초기 스탯은 다음 요소를 기반으로 계산한다.

```
최종 스탯

= 레벨 기본 스탯

+ Σ(종족 기본 스탯 × 종족 비율)

+ 향후 장비 및 버프 효과
```

예시

```
Human (50%)
ATK +10

Elf (50%)
ATK +4

최종 종족 보정
= 10 × 0.5 + 4 × 0.5
= 7
```

---

# 5. 직업 시스템

캐릭터 생성 시 기본 직업을 자동으로 부여한다.

| 항목    | 내용       |
| ----- | -------- |
| 초기 직업 | Beginner |
| 변경 여부 | 가능       |
| 활성 직업 | 하나만 활성화  |

향후 레벨이나 조건을 만족하면 새로운 직업을 획득할 수 있다.

---

# 6. 스킬 시스템

캐릭터는 종족 및 직업에 따라 초기 스킬을 지급받는다.

예시

| 종족    | 기본 스킬        |
| ----- | ------------ |
| Human | Slash        |
| Elf   | Magic Arrow  |
| Orc   | Power Strike |

직업 변경 및 레벨 상승에 따라 추가 스킬을 습득할 수 있다.

---

# 7. 캐릭터 선택 시스템

로그인 후 여러 캐릭터 중 하나를 현재 플레이 캐릭터로 선택한다.

```
계정
 ├── Knight
 ├── Wizard   ← 선택
 └── Archer
```

선택된 캐릭터는 `active = true`로 설정된다.

### 제약 사항

* 하나의 계정에서 동시에 하나의 캐릭터만 활성화 가능
* 선택 시 기존 활성 캐릭터는 자동 비활성화

---

---

# 9. 공개 프로필 기능

사용자는 자신의 캐릭터 공개 여부를 설정할 수 있다.

## 공개 시 표시

* 캐릭터 이름
* 종족 구성
* 직업
* 레벨

## 비공개 시 숨김

* 인벤토리
* 장비 정보
* 세부 스탯
* 퀘스트 진행 상황

---

# 10. 캐릭터 검색 기능

다른 플레이어는 공개된 캐릭터만 검색할 수 있다.

## 검색 조건

* 캐릭터 이름
* 직업
* 종족
* 레벨 범위

검색 결과에는 공개 정보만 표시한다.

---

# 11. 삭제 기능

캐릭터 삭제는 소유자만 가능하다.

삭제 전 확인해야 할 사항

* 본인 소유 캐릭터인지 확인
* 현재 로그인한 계정인지 확인
* 관련 데이터 삭제 정책 확인

삭제 후에는 복구할 수 없다.

---

# 12. 권한 및 예외 처리

## 일반 사용자

가능

* 자신의 캐릭터 생성
* 자신의 캐릭터 조회
* 자신의 캐릭터 선택
* 자신의 캐릭터 삭제
* 공개 여부 변경

불가능

* 다른 사용자의 캐릭터 수정
* 다른 사용자의 캐릭터 삭제
* 비공개 캐릭터 조회

---

## 관리자(Admin)

가능

* 전체 캐릭터 조회
* 전체 캐릭터 검색
* 비공개 캐릭터 조회
* 관리 목적의 정보 확인

---

# 13. 추천 API 목록

## 캐릭터 생성

```
POST /characters
```

---

## 내 캐릭터 목록

```
GET /characters/me
```

---

## 캐릭터 선택

```
PATCH /characters/{char_id}/select
```

---

## 캐릭터 상세 조회

```
GET /characters/{char_id}
```

---

## 캐릭터 스탯 조회

```
GET /characters/{char_id}/stats
```

---

## 캐릭터 스킬 조회

```
GET /characters/{char_id}/skills
```

---

## 공개 여부 변경

```
PATCH /characters/{char_id}/visibility
```

---

## 공개 캐릭터 검색

```
GET /characters/search
```

---

## 캐릭터 삭제

```
DELETE /characters/{char_id}
```

# 14. 구현 우선순위

1. 캐릭터 생성
2. 종족 선택 및 비율 설정
3. 종족 기반 초기 스탯 계산
4. 기본 직업 및 기본 스킬 지급
5. 캐릭터 선택 기능
6. 캐릭터 상세 정보 조회
7. 공개 프로필 기능
8. 공개 캐릭터 검색
9. 캐릭터 삭제
10. 인벤토리 및 아이템 시스템 연동

# 15. 기대 효과

* 데이터베이스에 저장된 종족 정보를 활용한 유연한 캐릭터 생성
* 단일 종족뿐 아니라 복합 종족(혼혈) 캐릭터 지원
* 종족 비율에 따른 동적 스탯 계산
* 계정별 캐릭터 관리 및 선택 기능 제공
* 공개 프로필과 검색 기능을 통한 RPG 커뮤니티 요소 구현
* 이후 인벤토리, 장비, 퀘스트, 크래프팅 시스템과 자연스럽게 연동 가능한 확장성 확보
